# Contract

## Published
- Contract ID: ABI-LIFECYCLE-V1
- Version: V1
- Status: Frozen
- Owner Agent: agent-1

- Scope: C ABI runtime/session/generation/checkpoint lifecycle semantics for `include/engine/*.h` and `src/engine/api.cpp`.
- Files:
  - `include/engine/api.h`
  - `include/engine/runtime.h`

  - `include/engine/status.h`
  - `src/engine/api.cpp`
  - `src/runtime/**`
  - `tests/engine/api_tests.cpp`
- Guarantees:
  - All public API entry points are C-compatible (`extern "C"`) and use fixed-width integer types at ABI boundaries.
  - `engine_api_version()` returns `100` for this baseline.
  - `engine_ping(nullptr)` returns `ENGINE_STATUS_INVALID_ARGUMENT`; non-null returns `ENGINE_STATUS_OK`.
  - Runtime lifecycle:
    - `engine_runtime_create` requires non-null config/output and `abi_version > 0`.
    - `engine_runtime_boot` is idempotent (`OK` when already booted).
    - `engine_runtime_shutdown` is idempotent when already shutdown, but returns `NOT_READY` while sessions are active.
    - `engine_runtime_destroy` returns `NOT_READY` while sessions are active; if booted and no sessions, it performs shutdown then destroys.
  - Session lifecycle:
    - `engine_session_create` requires booted runtime.
    - `engine_session_open`/`engine_session_close` are idempotent.
    - `engine_session_destroy` closes an open session (and active generation) before releasing resources and decrements runtime active-session count.
  - Generation lifecycle:
    - `engine_generation_start` requires open session, non-null request/prompt, and `max_new_tokens > 0`.
    - `engine_generation_step` requires active generation and returns monotonically increasing token IDs starting at `1`.
    - Generation finalizes automatically at token limit; `engine_generation_cancel` returns `NOT_READY` when generation is not active.
  - Checkpoint lifecycle:
    - `engine_state_checkpoint_save/load` require open session, non-empty path, and no active generation.
    - Save/load return `NOT_READY` while generation is active.
  - Invalid handle or missing required pointer arguments return `ENGINE_STATUS_INVALID_ARGUMENT`.
  - Ordering/state violations return `ENGINE_STATUS_NOT_READY`.
- Non-goals:
  - No persistence format or cross-process handle compatibility guarantee in V1.
  - No guarantee of stable internal state-machine names/telemetry fields.
  - No threading/concurrency safety guarantee.
